# 系统配置

## 分组说明
本分组包含用于系统配置管理的API接口，包括资产管理、白名单管理、系统状态查询等。属于平台管理类接口。

## 业务意图
- **主要意图**: 管理系统配置、维护资产信息、配置白名单
- **适用场景**: 平台运维、资产管理、白名单维护
- **数据流向**: 包含读写操作

## 包含接口

| 接口 | 功能描述 | 操作类型 |
|------|---------|---------|
| asset/add | 添加资产配置 | 写入 |
| asset/delete | 删除资产配置 | 写入 |
| system/status | 查询系统运行状态 | 只读 |
| whitelist/add | 添加白名单规则 | 写入 |
| whitelist/delete | 删除白名单规则 | 写入 |
| whitelist/list | 查询白名单列表 | 只读 |

## AI使用指南

### 何时使用
- 用户请求"添加资产"、"配置白名单"、"系统状态"
- 进行平台运维配置时
- 需要将正常业务加入白名单时

### ⚠️ 重要提醒
**白名单配置会影响检测效果，需谨慎操作！**

### 典型调用场景
```
场景1: 资产管理
- asset/add: 添加新资产（IP/CIDR、名称、类型）
- asset/delete: 删除资产

场景2: 白名单管理
- whitelist/add: 添加白名单（IP/域名/URL）
- whitelist/list: 查看现有白名单
- whitelist/delete: 移除白名单

场景3: 系统监控
- system/status: 获取系统运行状态
```

### 注意事项
1. 资产配置影响告警关联和资产分组
2. 白名单会跳过检测，确保是可信对象
3. 删除操作需确认影响范围
4. 建议定期审计白名单配置
