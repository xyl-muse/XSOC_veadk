# 白名单管理

## 分组说明
本分组包含用于管理白名单规则的API接口，支持白名单的增删改查操作。白名单配置会影响告警检测效果。

## 业务意图
- **主要意图**: 管理白名单规则、配置检测例外
- **适用场景**: 误报处理、正常业务加白、合规豁免
- **数据流向**: 包含读写操作

## 包含接口

| 接口 | 方法 | 功能描述 | 操作类型 |
|------|------|---------|---------|
| /api/xdr/v1/whitelists | POST | 新增白名单规则 | 写入 |
| /api/xdr/v1/whitelists | DELETE | 删除白名单规则 | 写入 |
| /api/xdr/v1/whitelists/{id} | PUT | 修改白名单规则 | 写入 |
| /api/xdr/v1/whitelists/list | POST | 查询白名单列表 | 只读 |

## AI使用指南

### 何时使用
- 用户请求"添加白名单"、"加白"、"配置豁免规则"
- 需要管理误报规则时
- 正常业务需要排除检测时

### ⚠️ 重要提醒
**白名单配置会影响检测效果，需谨慎操作！**

### 调用前确认清单
```
1. 确认加白对象：IP/域名/URL是否正确？
2. 确认加白原因：是正常业务还是误报？
3. 评估影响范围：加白后是否会漏报真实威胁？
4. 设置有效期：建议设置合理的过期时间
```

### 典型调用场景
```
场景1: 新增白名单
- 调用 whitelists (POST)
- 参数: 白名单类型、规则内容、描述等

场景2: 查询白名单列表
- 调用 whitelists/list (POST)
- 参数: 筛选条件、分页参数

场景3: 修改白名单
- 调用 whitelists/{id} (PUT)
- 参数: id(路径参数), 修改内容

场景4: 删除白名单
- 调用 whitelists (DELETE)
- 参数: 要删除的白名单ID列表
```

### 注意事项
1. **添加白名单前确认不会遗漏真实威胁**
2. 建议为白名单填写详细的描述和原因
3. 定期审计白名单规则，清理过期的规则
4. 白名单规则变更需记录操作日志
5. 敏感对象加白需要审批流程
