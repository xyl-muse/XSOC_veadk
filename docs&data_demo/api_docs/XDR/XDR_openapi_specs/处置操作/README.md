# 处置操作

## 分组说明
本分组包含用于执行安全处置操作的API接口，包括告警处置、事件处置、白名单状态管理等。是安全运营的核心动作类接口。

## 业务意图
- **主要意图**: 执行处置动作、更新处置状态
- **适用场景**: 应急响应、告警处置、事件处理、白名单管理
- **数据流向**: 写入操作，会修改系统状态

## 包含接口

| 接口 | 方法 | 功能描述 | 操作类型 |
|------|------|---------|---------|
| /api/xdr/v1/alerts/dealstatus | POST | 批量修改安全告警处置状态 | 写入 |
| /api/xdr/v1/alerts/dealstatus/list | POST | 查询告警处置状态列表 | 只读 |
| /api/xdr/v1/incidents/dealstatus | POST | 批量修改安全事件处置状态 | 写入 |
| /api/xdr/v1/incidents/dealstatus/list | POST | 查询安全事件处置状态列表 | 只读 |
| /api/xdr/v1/whitelists/{id}/status | PUT | 修改白名单启用状态 | 写入 |

## AI使用指南

### 何时使用
- 用户请求"处置告警"、"处理事件"、"修改处置状态"
- 批量更新告警或事件的处置状态时
- 启用/禁用白名单规则时

### ⚠️ 重要提醒
**本分组包含写入操作，执行前必须确认用户意图！**

### 调用前确认清单
```
1. 确认处置对象：告警UUID或事件UUID是否正确？
2. 确认处置动作：处置中、已完成、误报？
3. 确认影响范围：批量处置需确认范围
4. 记录处置原因：建议填写处置备注
```

### 典型调用场景
```
场景1: 处置告警
- 调用 alerts/dealstatus
- 参数: uuIds[], dealStatus, dealAction

场景2: 处置安全事件
- 调用 incidents/dealstatus
- 参数: uuIds[], dealStatus, dealAction

场景3: 启用/禁用白名单
- 调用 whitelists/{id}/status (PUT)
- 参数: id(路径参数), status
```

### 处置状态说明
- 1: 待处置
- 2: 处置中
- 3: 处置完成

### 注意事项
1. **必须确认用户意图后再执行写入操作**
2. 批量处置时确认UUID列表正确
3. 建议记录处置原因和操作人
4. 修改白名单状态会影响检测效果
